## Overview

`tmux-cli` is a command-line utility that enables Claude Code and other CLI agents to control applications running in tmux panes and windows. The tool automatically adapts its behavior based on whether you're operating inside or outside a tmux session.

## Core Functionality

The tool provides the ability to initiate, communicate with, monitor, and terminate programs operating within tmux panes. This allows agents to manage multiple terminal applications concurrently through a coordinated workflow.

### Mode Selection

- **Local Mode**: When operating inside tmux, it manages panes within your current window
- **Remote Mode**: When outside tmux, it establishes a separate tmux session with dedicated windows

## Available Capabilities

- Initiate shells and programs in new panes
- Transmit keystrokes with adjustable timing and automatic retry functionality
- Extract and review pane content
- Pause execution until a pane becomes inactive
- Execute commands and retrieve exit codes in JSON format
- Send interrupt signals (`Ctrl+C`) or escape sequences
- Built-in protection prevents deletion of your active pane

## Installation

The tool comes included with the `claude-code-tools` package through standard installation procedures.

## Basic Usage Pattern

**Step 1**: Always launch a shell first to prevent losing output when commands fail

**Step 2**: Transmit commands to the shell pane

**Step 3**: Monitor completion and extract results

**Step 4**: Terminate the pane when finished

## Example Workflow

```
tmux-cli launch "zsh"              # Initialize shell
tmux-cli send "pytest tests/" --pane=2    # Execute test suite
tmux-cli wait_idle --pane=2        # Hold until completion
tmux-cli capture --pane=2          # Retrieve output
tmux-cli kill --pane=2             # Cleanup
```
